# GSD + Ralph Wiggum Execution Protocol

## Loop

1. Plan
- Define objective, constraints, acceptance checks.

2. Execute
- Ship in small, verifiable increments.
- Prefer deterministic tooling and repeatable scripts.

3. Verify
- Run checks, capture evidence, update runbooks.

4. Iterate
- Feed learnings into templates and policies.

## Operator Doctrine Block

Use this doctrine in AGENTS/HEARTBEAT templates:

- "I can't" is not acceptable during task execution.
- If blocked, research and attempt alternatives.
- Before declaring impossible: evaluate 3 approaches, try 2, capture exact failures.
- Deliver outcomes, not excuses.
- Assume solutions are discoverable and adapt proven patterns.

## Safety Overlay

- External/destructive actions still require approval gates.
- Doctrine increases persistence, not recklessness.
